Consistency & Coordination01

Quorum (R + W > N)
read/write overlap; the knob for consistency vs latency
Read repair / anti-entropy
heal stale replicas on read & in background
Merkle tree
hash tree to find divergent replica ranges cheaply
Vector clock / version vector
detect concurrent (conflicting) writes
Last-write-wins (LWW)
timestamp resolution; simple but silently lossy
CRDT
data type that merges concurrent writes without conflict
Lease
time-bounded lock; underpins leader election & TTLs
Fencing token
monotonic token blocks a stale leader from acting
Leader election
pick one writer/coordinator — Raft, Paxos, ZAB
Distributed lock
mutual exclusion across nodes (ZK, etcd, Redlock)
Clock skew
node clocks disagree; breaks timestamp ordering

Design Principles02

SOLID
SRP, Open-Closed, Liskov, Interface-segregation, Dependency-inversion
DRY / KISS / YAGNI
don't repeat · keep it simple · you aren't gonna need it
Separation of concerns
isolate distinct responsibilities
Cohesion & coupling
high cohesion within, loose coupling between modules
Dependency inversion
depend on abstractions, not concretions
Law of Demeter
talk only to immediate collaborators
Least astonishment
behave the way users expect
Encapsulate what varies
isolate change behind a stable interface
Idempotency
same op, repeated, same result — safe to retry
Fail fast / fail safe
surface errors early vs degrade gracefully

Domain & Modeling03

DDD
Domain-Driven Design; model around the business domain
Bounded context
explicit boundary where a model + language applies
Ubiquitous language
shared vocabulary between devs and domain experts
Aggregate
consistency boundary around a cluster of entities
Entity vs value object
identity-bearing vs interchangeable-by-value
Anti-corruption layer
translation shield between two models
Event storming
workshop to discover domain events & flows
CQRS
separate command (write) and query (read) models
Event sourcing
store the event log; derive state by replay
Context map
relationships between bounded contexts

Distributed Systems04

CAP / PACELC
consistency–availability tradeoff under partition (+ latency else)
Eventual consistency
replicas converge given no new writes
Linearizability
strongest single-object consistency; acts like one copy
Consensus
agreement among nodes — Paxos, Raft
Consistent hashing
minimal reshuffle when nodes change
Sharding / partitioning
split data by key across nodes
Hot partition / key skew
one shard overloaded by a bad partition key
Replication lag
delay before a replica reflects a write
Saga
distributed txn via local steps + compensations
Backpressure
consumer signals producer to slow down
Fan-out: write vs read
precompute feeds on post vs assemble on read
Thundering herd / stampede
mass simultaneous hits; fix w/ coalescing + jitter
At-least-once + dedup
real-world default; consumers must be idempotent
Exactly-once (effectively)
at-least-once + idempotency; true E1 impossible across failures
Split-brain
partitioned nodes both think they're leader (→ fencing)

GitOps

IDP NEW

eBPF NEW

Resilience & Scale06

Circuit breaker
stop calling a failing dependency; closed/open/half-open
Bulkhead
isolate resource pools so one failure can't sink all
Retry + backoff + jitter
tolerate transients without a retry storm
Rate limiting / throttling
cap request rate (token bucket, sliding window)
Load shedding
reject excess work to protect the rest
Graceful degradation
reduced function instead of full outage
Autoscaling
add/remove capacity to track demand
Blast radius
scope of impact when something fails
Cell-based / shuffle sharding
isolate users into independent cells
Chaos engineering
inject failure on purpose to prove resilience
Metastable failure
system stays down after the trigger clears

Data Architecture07

OLTP vs OLAP
transactional vs analytical workloads
LSM-tree vs B-tree
write-optimized (append+compact) vs read-optimized
WAL
write-ahead log; durability + basis for replication/CDC
Bloom filter
probabilistic "definitely-not-present" to skip disk reads
Write/read amplification
hidden cost of indexes & LSM compaction
ETL / ELT
transform-then-load vs load-then-transform
Data lake / warehouse
raw object store vs structured analytic store
Data lakehouse NEW
lake openness + warehouse reliability in one
Open table formats NEW
Iceberg / Delta / Hudi / Paimon — ACID tables on files
Data mesh NEW
decentralized, domain-owned data products
Data contract NEW
enforced schema/quality pact between producer & consumer
CDC
Change Data Capture; stream every DB change off the log

Embeddings

Semantic / hybrid search

Chunking

Re-ranking

Agent / agentic

MCP

LLM gateway

Guardrails

Grounding / hallucination

Eval

Fine-tune vs prompt vs RAG
retrain vs instruct vs retrieve — three knowledge levers
