Practitioner's Field Notes

Term · gloss · ⚠ gotcha
the stuff you reason about on the job

Clocks, Ordering & Sequence01

Sequence number — monotonic counter giving a total order to events/writes.
Monotonic clock — never goes backwards; for measuring elapsed time.
LSN (Log Sequence Number) — position in the WAL; identifies each change.
Logical clock / Lamport — orders events without synced wall clocks.
Vector clock — per-node counters detecting concurrent writes.
Hybrid logical clock (HLC) — logical clock bounded to physical time.
Clock skew — node clocks disagree.

Cache & Write Strategies02

Cache-aside (lazy) — app reads cache, on miss loads DB & populates.
Read-through — cache itself loads from DB on miss.
Write-through — write to cache + DB synchronously.
Write-behind (write-back) — write cache now, flush to DB async.
Refresh-ahead — proactively reload hot keys before they expire.
Last-write-wins (LWW) — newest timestamp wins on conflict.
TTL & eviction (LRU/LFU) — expire by time; evict by recency/frequency under pressure.
Cache invalidation — remove/update stale entries on write.
Negative caching — cache "not found" results too.

Locking & Consistency03

Optimistic lock (version) — read version, write only if unchanged (CAS/ETag).
Pessimistic lock — lock the row up front for the duration.
ACID — atomicity, consistency, isolation, durability.
Read-your-writes — a user always sees their own last write.
Idempotency key — client token making a retried op safe.
Isolation levels — read-committed → repeatable-read → serializable.
Quorum (R + W > N) — overlapping read/write sets guarantee freshness.

Traffic Shaping & Routing04

Cell / POD architecture — independent stack per user subset.
Multi-region lag — async replication delay between regions.
Anycast / DNS PoP — same IP, routed to nearest point of presence.
GeoDNS / latency routing — DNS answers by client location/RTT.
L4 vs L7 load balancing — route on TCP/IP vs HTTP content.
LB algorithms — round-robin, least-connections, P2C (power-of-two).
Consistent-hash routing — sticky map of key→node for cache affinity.
Backpressure — downstream signals "slow down" upstream.
Load shedding — drop excess requests to protect the rest.
Circuit breaker — stop calling a failing dependency (open/half-open).
Admission control — gate requests at the door to protect SLOs.
Timeout + deadline propagation — cap wait; pass remaining budget down the chain.
Hedged request — fire a 2nd copy if the 1st is slow; take first to return.
Connection draining — stop new conns, let in-flight finish before shutdown.
Priority / QoS lanes — separate queues so critical traffic isn't starved.
DNS poisoning — attacker injects forged DNS records.

Top-Line Metrics05

P50 / P99 / P999 — latency at the 50th/99th/99.9th percentile.
Mean vs tail — average vs the slow extremes.
Uptime / availability — % of time the service meets its SLO.
Downtime budget — allowed unavailability (99.9% ≈ 43 min/mo).
Error budget — the SLO's allowed failure, spent on risk/velocity.
Saturation — how "full" the busiest resource is (USE method).

Capacity & Estimation05b

QPS / peak-to-average — request rate; peaks run 2–10× the average.
Little's Law — concurrency = arrival rate × latency.
Headroom — spare capacity kept above expected peak.

Scaling Failure Modes06

Retries (the dark side) — re-sending on failure to ride out transients.
Thundering herd — mass simultaneous requests on one resource.
Metastable failure — system stays down after the trigger clears.
Hot partition / key skew — one shard takes disproportionate load.
Cascading failure — one component's failure topples dependents.
Gray failure — degraded-but-alive; passes health checks while failing real work.
Coordinated omission — load tools under-count latency by pausing when the system stalls.

Deployment & Release07

Blue-green — two identical envs; flip traffic to the new one.
Canary — release to a small % first, watch metrics, then ramp.
Rolling — replace instances batch by batch.
Feature flag — decouple deploy from release; toggle at runtime.
Expand-contract migration — add new schema, migrate, drop old.

Messaging & Queues07b

At-least / at-most / exactly-once — delivery guarantees.
Dead-letter queue (DLQ) — sink for messages that keep failing.
Consumer lag — how far behind consumers are vs producers.
Ordering & partitions — order is per-partition, not global.
Outbox pattern — write event + data in one txn, relay publishes.

Debugging & Analytics08

Request tracing / request ID — correlation ID threaded through every hop.
Distributed tracing — spans across services (OpenTelemetry).
Debug / dynamic log levels — raise verbosity without redeploy.
Requests-as-data — emit each request as an analytics event.
Structured logging — logs as key-value/JSON, not free text.
Logs vs metrics vs traces — events · aggregates · request paths.

Counting at Scale09

Approx counting (batched) — buffer increments, flush aggregated.
Real-time counting — atomic increment per event (Redis).
Probabilistic (HLL / CMS) — distinct/frequency in tiny fixed memory.

Rate Limiting10

In-memory (per node) — each node enforces its own limit locally.
Centralized (Redis) — shared atomic counter for a true global limit.
Full-mesh / gossip — nodes share counts to approximate global.
Token bucket — tokens refill at rate R, burst up to B.
Sliding window — count over a moving interval, not a fixed reset.
Cost-based limiting — charge requests by weight, not count.

Authentication11

Token validation (JWT) — verify signature + claims, no DB hit.
JWKS — public-key endpoint for verifying JWT signatures.
OAuth2 + PKCE — delegated auth; PKCE secures public clients.
mTLS — mutual cert auth for service-to-service.
AuthN vs AuthZ — who you are vs what you may do (RBAC/ABAC/ReBAC).
Refresh-token rotation — issue a new refresh token on each use.

Costing & Billing12

No free lunch — every quality (a nine, low latency) has a cost.
Cost per customer at scale — unit economics of serving one user.
Pay-per-request pricing — bill per call when cost is predictable.
Pay-per-resource pricing — bill by compute/storage/egress consumed.

Isolation (Tenant & Regulated)13

Compute isolation — separate the execution environment per tenant (VM, dedicated node, namespace+cgroups).
Storage / DB isolation — silo data per tenant: row (shared), schema, or database-per-tenant.
Network isolation — per-tenant VPC/subnet, security groups, no shared egress.
Noisy neighbor — one tenant's load degrades others on shared infra.
Data residency — keep a tenant's data in a specific region/jurisdiction.
Single-tenant vs pooled — dedicated stack vs shared with logical separation.

Compliance14

GDPR — EU data-protection law: consent, access, and the right to be forgotten.
On-demand data deletion — purge a user's data across all stores on request (GDPR/CCPA "right to erasure").
HIPAA — US health-data rules: protect PHI, audit access, sign BAAs with vendors.
PCI-DSS — payment-card security standard for handling cardholder data.
SOC 2 — audited report on security/availability controls (trust for B2B).
Data classification — tag data by sensitivity (public → PII → PHI/PCI).
Audit trail — immutable, tamper-evident log of who accessed/changed what.

Operations & Incidents15

SRE — Site Reliability Engineering; run ops with software & SLOs, not toil.
Toil — manual, repetitive, automatable work with no lasting value.
On-call / runbook — rotation that responds; runbook = step-by-step fix guide.
Incident severity (SEV) — SEV1 (critical) → SEV3+; sets response & comms.
MTTD / MTTR / MTBF — mean time to detect / recover / between failures.
Blameless postmortem — write-up of cause & fixes, no finger-pointing.
Alert fatigue — too many low-value alerts dull response.
Change freeze — halt deploys during peak/risk windows.
Capacity / DR drill — rehearse failover & load before you need it.
Golden signals — latency, traffic, errors, saturation: what to watch.

Tech Finance & Business16

CapEx vs OpEx — upfront capital (own hardware) vs ongoing spend (cloud).
TCO — total cost of ownership: build + run + people + opportunity.
Unit economics — cost & revenue per unit (user, request, GB).
Gross margin — revenue minus cost of serving it (infra, support).
COGS — cost of goods sold; for SaaS = the cost to run the service.
Burn rate / runway — monthly net spend / months of cash left.
Reserved / spot / committed-use — discounts for upfront or interruptible capacity.
Chargeback / showback — bill (or just show) teams their cloud spend.
ROI / payback period — return vs cost; time to recoup an investment.
Build vs buy — in-house vs vendor/SaaS for a capability.
