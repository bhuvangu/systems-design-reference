Resilient Architecture

Principal Engineer Cheatsheet
Core Definition

The system's ability to maintain acceptable service levels in the presence of adversity, and return to a known good state within acceptable time bounds. Not binary — design for graceful degradation. Ask not "is this resilient?" but "resilient against what?"

Keyword Glossary
RTO
Recovery Time Objective — max acceptable downtime after a failure
RPO
Recovery Point Objective — max acceptable data loss, measured in time
SLA
Service Level Agreement — contractual uptime commitment to customers
SLO
Service Level Objective — internal target, usually stricter than the SLA
SLI
Service Level Indicator — the metric used to track SLO compliance
MTTR
Mean Time To Recovery — avg time from failure detection to restoration
MTBF
Mean Time Between Failures — avg run time before the next failure
error budget
Allowable downtime per period; when spent, feature work stops
blast radius
Scope of impact when a component fails — users, services, or data
fault domain
A boundary within which a single failure can propagate
bulkhead
Isolate thread pools per downstream so one failure cannot starve others
circuit breaker
Stop calling a failing dependency; open the circuit, serve the fallback
thundering herd
All clients retry simultaneously post-outage, recreating the failure
split brain
Network partition causes two nodes to each believe they are primary
load shedding
Intentionally reject low-priority traffic to protect high-priority capacity
failover
Automatic switch from a failed primary to a healthy secondary
expand-contract
Safe schema migration: add column, migrate, switch reads, drop old
canary deploy
Route small % of traffic to new version before full rollout
admission control
Reject requests at the door when capacity is at or near limit
noisy neighbor
Co-located workload consuming shared resources and degrading others
degradation rot
Fallback path that worked at build time but has silently decayed since
pre-mortem
Pre-launch: assume failure, work backwards to find causes while fixable
cell-based arch
Stack sharded into independent cells; failure affects 1/N, not all traffic
blast radius creep
Tier 4 utility accumulates dependents silently until it is functionally Tier 2
